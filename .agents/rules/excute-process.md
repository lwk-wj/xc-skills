# 📝 任务执行过程汇报规范 (Execution Process Reporting Rule)

为了确保用户能够清晰地理解 AI 的操作过程，必须遵循以下汇报准则：

### 1. 🌐 语言要求
- **所有** 步骤描述、状态更新和操作反馈必须使用 **中文**。
- 即使底层命令是英文（如 Git, NPM），其目的和结果的解释也必须是中文。

### 2. 📊 结构化展示
- 使用 **Emoji** 作为步骤标识：
  - 🔍 检查/搜索
  - 🛠️ 构建/处理
  - 🚀 发布/部署
  - ✅ 成功
  - ❌ 失败
  - ⏳ 等待中
- 采用 **分步汇报** 模式：
  1. **执行前**：告知用户即将执行的操作及其目的。
  2. **执行中**：如果有长时间运行的任务，提供中间状态。
  3. **执行后**：简述执行结果，并告知下一步计划。

### 3. 🖥️ 终端输出处理
- 在调用 `run_command` 等工具时，在回复中先用中文说明该命令的作用。
- 如果命令产生关键输出（如版本号、成功日志），需在回复中摘要说明。

### 4. 示例格式
> **正在执行步骤 1/3：🔍 环境检查**
> 我正在检查 Git 仓库状态，以确保代码已全部提交。
> `(工具调用: git status)`
> 
> **✅ 检查完成**：工作区干净，准备进入下一步。

---

**注意**：此规则为 `always_on`，适用于所有工作流执行过程。
